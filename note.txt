Q: ImageField raises “Incorrect string value” error when store an image having Chinese name to MySql db. 
A: 1. In Mysql, check column: SHOW COLUMNS FROM projects_image in linlinzhong_v1;
   2. Check settings for char in mysql: SHOW VARIABLES WHERE Variable_name LIKE 'character\_set\_%' OR Variable_name LIKE 'collation%';
   3. fix projects_image type: ALTER TABLE linlinzhong_v1.projects_image MODIFY COLUMN data VARCHAR(100) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL;
   4. optional step: update /etc/mysql/my.cnf: 
      	[client]
	default-character-set = utf8mb4

	[mysql]
	default-character-set = utf8mb4

	[mysqld]
	character-set-client-handshake = FALSE
	character-set-server = utf8mb4
	collation-server = utf8mb4_unicode_ci
   5. Helpful Links:
	https://mathiasbynens.be/notes/mysql-utf8mb4#character-sets
	http://stackoverflow.com/questions/2108824/mysql-incorrect-string-value-error-when-save-unicode-string-in-django
	http://dev.mysql.com/doc/refman/5.0/en/show-columns.html
	http://dev.mysql.com/doc/refman/5.0/en/charset-connection.html

Q:#Django# Context Stack
A: Create different levels when do push() and remove top levels when pop()
   E.g. from django.template import Context
	c = Context()
	c['a'] = 1
	c.push() # Make a new Context level
	c['a'] = 2
	print(repr(c)) # [{'a': 1}, {'a': 2}]
